# Інтелектуальна вебсистема автоматизованого підбору конфігурації персонального комп'ютера на основі потреб користувача

## Опис проєкту
Цей проєкт є демонстраційним прототипом вебсистеми, яка автоматично підбирає конфігурацію персонального комп'ютера відповідно до потреб користувача.

Система допомагає користувачу:
- визначити сценарій використання ПК;
- отримати рекомендовану конфігурацію;
- порівняти альтернативні варіанти збірки;
- переглянути пояснення щодо вибору комплектуючих;
- зберегти збірку в поточному профілі;
- переглянути історію запитів і повторно відкрити попередні результати.

Проєкт розроблено в межах бакалаврської роботи за темою:
**«Інтелектуальна вебсистема автоматизованого підбору конфігурації персонального комп'ютера на основі потреб користувача»**.

---

## Основні можливості
- вебінтерфейс для підбору конфігурації ПК;
- підтримка сценаріїв:
  - ігровий ПК;
  - офісний ПК;
  - ПК для навчання;
  - ПК для монтажу / 3D;
- локальне AI/ML-визначення призначення ПК на основі текстового опису;
- автоматичний та ручний режим підбору бюджету;
- формування 3 конфігурацій:
  - основна рекомендована;
  - 2 альтернативні;
- перевірка сумісності компонентів;
- пояснення вибору комплектуючих;
- підтримка профілів користувача;
- історія запитів;
- збережені збірки;
- автоматичне оновлення цін з зовнішнього джерела;
- unit-тести для основних сценаріїв.

## Як працює система

### 1. Вибір сценарію використання
Користувач обирає тип ПК або вводить текстовий опис своїх потреб. Якщо використовується текстовий опис, локальна ML-модель прогнозує найбільш імовірний сценарій використання.

### 2. Введення параметрів
Після вибору сценарію користувач задає бюджет, пріоритет підбору та додаткові параметри, наприклад:
- роздільну здатність;
- бажаний FPS;
- програми;
- кількість вкладок;
- кількість моніторів;
- обсяг RAM та SSD;
- тип графіки;
- наявність Wi-Fi.

### 3. Побудова конфігурації
Сервісний шар нормалізує вхідні параметри та передає їх у builder, який:
- визначає вимоги до збірки;
- обирає сумісні комплектуючі;
- перевіряє обмеження бюджету;
- формує основну рекомендацію;
- формує 2 альтернативні конфігурації.

### 4. Формування результату
Система показує:
- назву сценарію;
- рівень збірки;
- загальну вартість;
- усі комплектуючі;
- пояснення до кожного компонента;
- примітки щодо сценарію та бюджету.

### 5. Збереження та історія
Користувач може:
- зберегти збірку;
- переглянути попередні результати;
- повторно відкрити збірку з історії;
- видалити окремий запис історії;
- очистити всю історію.

## AI/ML-модуль
Локальний AI-модуль використовується для автоматичного визначення призначення ПК на основі текстового опису користувача.

Підтримувані класи:
- `gaming`
- `office`
- `study`
- `creator`

Особливості:
- модель працює локально;
- зовнішні API не використовуються;
- прогноз приймається за порогом впевненості;
- якщо впевненість недостатня, користувач може обрати сценарій вручну.

## Оновлення цін
Технічні характеристики компонентів зберігаються локально у `parts_db.py`, а ціни можуть оновлюватися автоматично через скрипти в папці `scripts/`.

Основні файли:
- `data/price_links.json` — джерела цін;
- `data/price_feed.json` — збережений актуальний набір цін;
- `scripts/refresh_prices.py` — оновлення цін.

### Приклад запуску оновлення цін
```powershell
python scripts/refresh_prices.py
```

## Запуск проєкту

### 1. Активувати віртуальне середовище
```powershell
.venv\Scripts\activate
```

### 2. Встановити залежності
```powershell
python -m pip install -r requirements.txt
```

Якщо `requirements.txt` ще не містить усіх пакетів, мінімальний набір для запуску:
```powershell
python -m pip install fastapi uvicorn jinja2 python-multipart httpx pytest
```

### 3. Запустити застосунок
```powershell
python -m uvicorn app:app --host 127.0.0.1 --port 8010
```

Після запуску застосунок буде доступний у браузері за адресою:
```text
http://127.0.0.1:8010
```
